# 03 — Network Binding and TLS

## Goal

Default the server to `127.0.0.1` and add optional TLS/HTTPS support.

## Context

OpenMuara's default deployment is local. Binding to `0.0.0.0` by default risks exposing the dashboard in shared environments. We change the default to `127.0.0.1` and make network exposure explicit.

TLS is optional but recommended in hardened mode.

## Required Output

1. Use existing `server.host` as the bind address (default `127.0.0.1`).
2. Add config fields:
   - `server.tls_cert`
   - `server.tls_key`
3. Update `cmd/muara/start.go` and `internal/server/server.go` to use `server.host` for binding.
4. Add TLS listener when both cert and key are configured.
5. `muara security gen-cert` will be implemented in prompt 06.
6. Add validation: warn if `server.host == 0.0.0.0` and auth/TLS are disabled.
6. Add tests for binding and TLS.
7. Update `TRACKING.md`, `DECISIONS.md`, `RISKS.md`, and `HANDOFF.md`.

## Decision Criteria

- Default bind is `127.0.0.1` via `server.host`.
- Docker/shared usage requires explicit `server.host: 0.0.0.0`.
- TLS is opt-in via cert/key paths.
- No runtime overhead when TLS is disabled.

## Quality Gate

- `go test ./...`
- Smoke test starts server on `127.0.0.1` by default.
- TLS tests use self-signed cert generated by CLI helper.
